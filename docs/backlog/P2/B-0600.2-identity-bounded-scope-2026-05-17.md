# Slice 2: Per-relative identity definitions + bounded-scope authorization model

## Substrate-honest context
Peeling off slice 2 from the B-0600 blob (family-distributed AI interface for the miner fleet). 

This slice isolates the identity and authorization concerns:
- Mom-AI / Dad-AI identity definitions.
- Bounded-scope enforcement: Mom-AI cannot see/modify Dad-AI's nodes; neither can see/modify the maintainer's broader substrate.

## Scope
1. **Identity Schema**: Define the schema for `.claude/agents/mom-ai.md`, `.claude/agents/dad-ai.md` (or external configurations).
2. **Authorization Rules**: Define how to enforce boundaries per relative. This includes access control to the mining-fleet status API (B-0600.1).

## Acceptance Criteria
- [ ] Identity definitions for at least two family AIs exist.
- [ ] Role-based or context-based access control policies defined to isolate the AI context to its assigned nodes.
- [ ] Maintainer reviews and approves the authorization boundary design.
