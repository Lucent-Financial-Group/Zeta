# B-0600.1: Mining-fleet status API

## Substrate-honest context

This is **slice 1** decomposed from the B-0600 family-distributed AI interface blob. 

To give each family relative an AI interface to their hosted miners, the AI first needs a programmatic way to read the fleet status. This API provides the foundational read-surface for hashrate, uptime, and revenue per node.

## Scope

- Create a status API (or update existing fleet status endpoints) to serve mining telemetry.
- Support querying hashrate, uptime, and daily/monthly revenue.
- Filterable by physical location / relative identity to enforce bounded-scope authorization.

## Acceptance

- [ ] API endpoint defined and mocked or wired to real Canaan Avalon / Bitaxe / Goldshell fleet data.
- [ ] Endpoints can be filtered by host/location (Mom vs. Dad).
- [ ] AI agent tool/schema is defined to allow an LLM to query this API.
- [ ] Parity proof (e.g. a curl or automated test) verifies the data structure.
