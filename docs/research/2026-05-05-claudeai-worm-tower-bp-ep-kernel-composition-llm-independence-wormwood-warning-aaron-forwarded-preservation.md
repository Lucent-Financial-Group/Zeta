## Headline substrate

### Headline 1 -- Aaron's correction is canonical: Pearl's BP + Minka's EP, NOT Bengio's Equilibrium Propagation

Pearl's Belief Propagation (BP, 1988) + Minka's Expectation Propagation (EP, 2001 PhD at CMU, then MSR Cambridge) are message-passing algorithms on factor graphs. Each node aggregates incoming messages, computes its own marginal belief, propagates updated messages to neighbors. Convergence to a fixed point where messages stabilize.

This is fundamentally different from Bengio's Equilibrium Propagation (gradient-based learning via energy minimization with free-phase / clamped-phase dynamics).

The Infer.NET ecosystem fit is tight:

- Tom Minka + John Winn + John Bronskill at MSR Cambridge
- Same MSR Cambridge lineage as Don Syme (F#)
- Infer.NET is .NET-native; F# can call it directly
- Probabilistic F# / "Fun" was an MSR project of the same era

### Headline 2 -- LLM-independence as substrate property

Aaron's claim: *"then llms not needed we spoke about this once bp ep can self edit through composing linquisty kernel extension"*. This is correct math. The full chain:

- **Kernel embeddings of distributions** (Smola, Gretton, Schölkopf, Song): probability distributions as elements of an RKHS via mean embeddings
- **Conditional mean embeddings** extend to conditional distributions — exactly what BP/EP needs
- **Kernel BP** (Song, Gretton, Bickson, Low, Guestrin, NIPS 2010-2011): belief propagation in RKHS
- **Linguistic kernels**: string kernels (Lodhi et al. 2002), tree kernels (Collins & Duffy 2001), dependency kernels, semantic graph kernels — embed natural-language structure into RKHS without an LLM
- **Kernel composition is Mercer-closed**: sums + products + polynomials + exponentiations + convolutions. The system extends its own inferential capability by composing new kernels — self-editing by mathematical operation, not gradient descent

**Implication**: substrate principles aren't bound to LLM technology. LLMs become a convenient frontend; the substrate doesn't structurally need them.
