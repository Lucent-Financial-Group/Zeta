### Headline 3 — DeepSeek V4 with CSA+HCA attention

V4-Pro: 1.6T total / 49B active. V4-Flash: 284B total / 13B
active. Both released April 22-24 2026, MIT-licensed, 1M context
native, switchable Thinking/Non-Thinking modes (folds R line in).

CSA+HCA attention architecture (NOT "DSA" — correct the earlier
AI's framing):

- **Compressed Sparse Attention (CSA)**: compact KV with top-k
  sparse selector
- **Heavily Compressed Attention (HCA)**: folds many tokens
  into single entries
- Interleaved across layers

Performance vs V3:

- 90% KV cache reduction
- 73% per-token FLOPs reduction
- At 1M context, V4-Pro: 27% V3.2 FLOPs, 10% V3.2 KV; V4-Flash:
  10% FLOPs, 7% KV

**Architectural path vs compression-on-top path** is the
substantive divergence from Google. TurboQuant compresses an
existing KV cache post-hoc; DeepSeek redesigns attention so the
cache is structurally smaller from the start. **Different layer
of the stack — they compose, they don't compete. Run V4 +
TurboQuant on top of CSA+HCA and you stack the wins.**

Composition with Zeta architecture (CSA+HCA shape):

- Sparse selectors = filter operators (signed Z-set restriction)
- Compressed entries = aggregation operators (sum/fold)
- Interleaved layers = sequence of incremental rewrites
- Switchable Thinking/Non-Thinking = mode-conditioned dataflow
  branching

This is a stronger compositional fit than TurboQuant's post-hoc
KV cache compression. CSA+HCA could land in the algebra itself,
not just the runtime layer.
