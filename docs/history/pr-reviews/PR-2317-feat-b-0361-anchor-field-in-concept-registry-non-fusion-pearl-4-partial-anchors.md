## Description

## Summary

- Adds `anchor?: string` and `anchorState?: AnchorState` fields to the `Concept`
  interface in `tools/alignment/concept_registry.ts` (schema bumped to v1.1)
- Introduces a `KNOWN_ANCHORS` static map linking concept IDs to their established
  academic/formal definitions — O(1) lookup, no runtime deps, additive-only change
- Adds `anchoredCount: number` to `Registry` as a lightweight coverage metric
- Five concepts now carry anchors in the output:
  - **non-fusion** (anchored): Pearl (2009) d-separation / interventional independence
  - **bidirectional-alignment** (partially-anchored): Russell 2019 CIRL
  - **retraction-native** (partially-anchored): Shapiro et al. 2011 CRDTs
  - **glass-halo** (partially-anchored): Lipton 2018 interpretability tradition
  - **no-directives** (partially-anchored): Korsgaard 1996 self-constitutive agency

## What this prevents

The B-0361 origin: Claude.ai adversarial review demonstrated that the Z3 tautology
catches (shadow catch #30) happen when a property is proven without first checking
whether it already has an established formal definition. Non-fusion was the canonical
example — it maps directly to Pearl's interventional independence d-separation
criterion. Without the anchor, you write a proof of something trivially true by
definition; with the anchor, you cite rather than reinvent.
