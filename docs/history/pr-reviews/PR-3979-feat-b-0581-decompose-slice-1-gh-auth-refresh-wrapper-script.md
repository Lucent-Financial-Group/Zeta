- **`Copilot` (bot)** at 2026-05-19T00:58:05Z on `tools/auth/gh-auth-refresh-wrapper.ts`:30 (association: NONE)

  <pre>
  A single `TextDecoder` instance is created at line 19 and shared between the two concurrent `handleOutput` invocations (one for stdout, one for stderr). `TextDecoder` is stateful: it retains partial multi-byte sequences between `decode()` calls. When two concurrent async loops feed chunks from different streams into the same decoder, partial bytes left from one stream can be wrongly prepended to the next chunk of the other stream, producing corrupted output and potentially breaking the `PROMPT.includes` / one-time-code regex matching. Move the `TextDecoder` allocation inside `handleOutput` so each stream gets its own decoder instance.
  </pre>
