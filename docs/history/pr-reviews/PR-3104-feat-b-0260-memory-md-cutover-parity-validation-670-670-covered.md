fix(b-0260): broaden link-extraction regex to include path separators

The previous pattern /\([a-zA-Z0-9_\-\.]+\.md\)/ excluded '/', silently

](memory/observed-phenomena/foo.md). This under-counted totalOldIndexEntries

- Use a capture group instead of slice(1,-1) for the extracted path
- Strip leading "memory/" prefix (validator joins with MEMORY_DIR anyway)
- Skip known non-memory repo prefixes (docs/, tools/, etc.)
- Skip prose false-positives where an intermediate path component ends
  with .md (e.g. "AGENTS.md/GEMINI.md" inside parenthetical prose)
