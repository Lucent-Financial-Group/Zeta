## What the script does

`check-orchestrator-state.ts` emits structured JSON:
- `currentBranch` / `expectedBranch` / `branchMatch` (same as verify-branch)
- `dirtyFiles` — `git status --short` lines
- `worktrees` — parsed `git worktree list --porcelain` blocks
- `driftedWorktrees` — worktrees (other than the caller's own) on `ZETA_EXPECTED_BRANCH` — flags the CWD-bleed-over hazard

Exit 0 = clean; exit 1 = branch mismatch or drifted worktrees.

  **&lt;sub&gt;&lt;sub&gt;![P1 Badge](https://img.shields.io/badge/P1-orange?style=flat)&lt;/sub&gt;&lt;/sub&gt;  Exclude caller worktree instead of slicing from second entry**

  `checkOrchestratorState` assumes the first `git worktree list --porcelain` record is the caller’s worktree, but Git documents that the first record is the **main** worktree. When this script runs from a linked worktree with `ZETA_EXPECTED_BRANCH` set, `worktrees.slice(1)` includes the caller itself, so `driftedWorktrees` can falsely contain the current worktree and the script exits `1` even with no real drift. This makes the AC5 check fail in common multi-worktree sessions; exclude by matching the current working directory path instead of positional slicing.
