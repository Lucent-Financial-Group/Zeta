## Summary

Per Aaron 2026-05-15T~15:25Z (after observing the 2-hour runaway grep): *"references/upstreams/ in code we ignore this folder everywhere casue its not our code but other githubs we reference for ideas"*

New auto-loaded rule that encodes the search-hygiene discipline so future Zeta AIs don't hit the same trap.

## The failure mode

`references/upstreams/*` is gitignored (regeneratable mirror state). Git operations skip it. But plain `grep -r` / `find | xargs grep` don't — they walk the entire filesystem. My manifesto-search one-liner walked the upstream tree (protobuf docs, gRPC tests, Redis manifests, etc.) and ran for 2+ hours until Aaron's "11 shells running is impressive otto" observation surfaced it.

## The discipline

1. Prefer `rg` (ripgrep — respects gitignore by default)
2. Plain `grep` needs explicit `--exclude-dir=references/upstreams`
3. `find | xargs grep` needs `-not -path './references/upstreams/*'` filter BEFORE xargs
4. Explicit allowlists beat exclude lists — name `memory/`, `docs/`, `.claude/`, `tools/` instead of starting from `.`
5. When `references/upstreams/` IS the right search target, EXPLICITLY target the subtree
