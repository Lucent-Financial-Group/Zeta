  <pre>
  `extractTimestamp` is run on `context` that prepends the previous non-empty line. If both lines contain dates (common in logs), `DATE_RE.exec(context)` returns the first date (from the previous line), so later instructions can get the wrong timestamp (e.g., a 2026-05-02 line would be tagged 2026-05-01). Prefer extracting from the current `raw` line first (or choose the last date match in context) and only fall back to nearby context when `raw` has no date.
