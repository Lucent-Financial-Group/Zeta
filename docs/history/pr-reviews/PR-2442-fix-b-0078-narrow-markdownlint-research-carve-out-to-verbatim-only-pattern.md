## Description

## Summary

- Narrows `.markdownlint-cli2.jsonc` ignore pattern from `docs/research/2026-*-*.md` (all date-prefixed research docs) to `docs/research/2026-*verbatim*.md` (only files that explicitly signal verbatim content via the word "verbatim" in the filename).
- Addresses Codex P1 finding from PR #663 (tracked as B-0078): the broad pattern was silently skipping lint for all date-prefixed research docs, including author-controlled notes that should be linted.
- Updates the backlog row (B-0078) to `status: resolved` with pre-start checklist and audit result.

## Audit result

- 219 existing date-prefixed research files linted against the project's disabled-rules profile (MD013/MD031/MD033/MD034/MD036/MD040/MD004/MD041 off) — **exit 0, no violations**.
- No file renames required to avoid lint failures.
- 32 files already contain "verbatim" in the filename and remain covered by the narrowed carve-out.
- ~187 files that are author-controlled or auto-lintable now gain lint coverage going forward.

## Convention going forward

New verbatim ferry absorbs should include `-verbatim` in the filename  
(e.g. `2026-05-10-amara-topic-verbatim-aaron.md`) to opt into the carve-out.  
This matches the naming already used by most recent verbatim absorbs in the repo.
