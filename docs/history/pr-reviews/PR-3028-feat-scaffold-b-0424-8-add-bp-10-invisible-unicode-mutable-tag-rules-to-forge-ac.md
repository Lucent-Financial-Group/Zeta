The literal invisible codepoints (U+200B–U+2069) in the pattern-regex
character class caused the invisible-unicode-in-text rule to fire on
its own source file when semgrep scanned **/*.yml targets — a bootstrap
paradox where the rule detected itself.

Root .semgrep.yml avoids this by storing \uXXXX as ASCII YAML escape
sequences; YAML expands them to real codepoints at parse time so the
regex works, but the file on disk is invisible-char-free.

Apply the same pattern to both scaffold templates.
