B-0168 acceptance criteria requires worked translations for six real Lucent/Zeta situations. Three were already done (PR-review critique, security-incident disclosure, recruiting offer+filter). This PR delivers the fourth: **partner integration discussion**.

### Why this content shape is architecturally interesting

The layer-selection algorithm fires in tension with itself here:
- **Q1** ("Who is structurally in the audience?"): partner company = external → Beacon-safe or higher.
- **Q3** ("What register has the audience opted into?"): corporate partner-company contacts = Professional register by default.

The resolution — **Professional for Lucent-attributable partner communication** — is the correct one, and this is the first worked translation where the tension between Q1 and Q3 must be explicitly resolved. The prior translations (recruiting, security incident, PR review) all have unambiguous Q1/Q3 alignment.

The translation also demonstrates how the **pirate-not-priest discipline operates in Professional layer through structure rather than vocabulary**: the proposal explicitly surfaces the integration prerequisite and assigns it to the correct party *before* listing benefits. That ordering IS the discipline at the Professional-layer vocabulary register.
