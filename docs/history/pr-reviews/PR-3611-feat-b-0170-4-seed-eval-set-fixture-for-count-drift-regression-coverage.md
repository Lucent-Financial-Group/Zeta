    <pre>
    Addressed in commits e0b812f + 5d1d6ad.

    The HTML provenance comment originally restated `"9 drift instances"`, producing two matching findings (one from the comment, one from the body). The `length &gt;= 1` + `findings[0]` assertion could be satisfied by the comment alone, masking regressions in body-claim detection — exactly the concern you raised.

    Two-part fix:

    1. **Fixture rephrased** (`fixtures/count-drift-9-vs-15.md`): the HTML provenance comment no longer restates the `&lt;number&gt; &lt;noun&gt;` pair. An explanatory `NOTE:` section documents why, citing this thread. Body claim + 15-row table preserved unchanged.

    2. **Test tightened** (`fixtures.test.ts`): `expect(findings.length).toBeGreaterThanOrEqual(1)` → `expect(findings.length).toBe(1)` plus `expect(finding.line).toBe(24)` pinning the body-claim line. A regression in body-claim detection can no longer be masked by an HTML-comment match.

    Empirically verified locally:

    ```

    1 count-drift finding(s).
    ```

    Exactly one finding, on the body line.
