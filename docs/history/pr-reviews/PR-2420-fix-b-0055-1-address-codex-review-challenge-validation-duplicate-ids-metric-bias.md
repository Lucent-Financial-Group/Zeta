## Summary

Follow-up to #2419 which merged before these Codex review fixes could land. Addresses all three threads from the automated Codex review:

- **P2**: `validateFlag` now requires non-empty `challengeHistory` for `challenged` state (not just `defended`/`superseded`) — a flag can no longer be `challenged` with zero challenge records
- **P2**: `validateCatalog` now detects duplicate `CtfFlag.id` values across entries, enforcing the ECF-NNN uniqueness invariant at the catalog boundary  
- **P1**: `meanDaysFlagPlantedToFirstChallenge` metric now measures from `stakeDate` to the **first filed challenge** (any outcome, including `open`) rather than first resolved challenge — eliminates the high-bias that would occur once real flags enter `challenged` state
