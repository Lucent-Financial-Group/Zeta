## Description

## Summary

Setup script that registers two parallel background loop agents as launchd services:

- **com.zeta.claude-loop-opus** — Opus model, separate worktree at `~/.local/share/zeta-claude-loop-opus/Zeta`
- **com.zeta.claude-loop-sonnet** — Sonnet model, separate worktree at `~/.local/share/zeta-claude-loop-sonnet/Zeta`

Each gets its own state dir, log dir, and lock dir. Both use the same tick script (`claude-loop-tick.ts`) with `ZETA_CLAUDE_LOOP_MODEL` set differently. Both write to model-ratings.jsonl. They repair each other — if one agent's PR gets review findings, the other can drain them.

Unloads the old single-agent service (`com.zeta.claude-loop`) on setup.

### Why parallel not sequential

Sequential A/B (week of Opus, week of Sonnet) has a confound: backlog state differs each week. Parallel A/B produces fair comparison — same backlog, same time window, different model. The output difference IS the model difference.

### Usage

```bash
bun tools/ops/setup-dual-background-agents.ts          # register both
bun tools/ops/setup-dual-background-agents.ts --dry-run # preview
bun tools/ops/model-rating-report.ts --reviews          # compare results
```
