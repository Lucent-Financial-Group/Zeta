## Description

## Summary

- Implements `tools/playwright/github-ui/drain-log.ts` — the append-only JSONL reversibility substrate required by B-0064 Phase 2
- Every UI mutation produces a `MutationLogEntry`; `appendEntry()` writes it to `docs/hygiene-history/playwright-mutations/log.jsonl`
- `listPending()` returns all entries whose effective status is `"applied"` (latest-record-per-id reduces over the append-only file)
- `revert(entryId)` reads the inverse action, calls `mutate()` with it, then appends a `status: "reverted"` marker — original line untouched (audit trail preserved)
- Adds `docs/hygiene-history/playwright-mutations/README.md` explaining the JSONL semantics

## Design choice: append-only revert event

Reverts are stored as a second line with the same `id` and `status: "reverted"`. This avoids any read-modify-write cycle on the log file, making it crash-safe. `listPending()` computes current state by reducing to the latest-status-per-id.

## Remaining done-criterion (follow-on slice)

Wire `appendEntry()` into the `mutate()` call site so entries are written automatically on every real mutation. That coupling was explicitly deferred from B-0321 per the backlog split.
