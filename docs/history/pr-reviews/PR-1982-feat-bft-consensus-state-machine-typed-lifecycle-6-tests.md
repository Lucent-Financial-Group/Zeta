  In the `Voting, CastVote` transition, the incoming `value` is accepted without checking it against `state.Proposal`. That allows a round to `Propose("A")` but later collect a quorum for `"B"`, and `Finalize` will commit `"B"` even though it was never proposed in this round. This breaks the round invariant implied by `Proposal`/`Proposer` and can produce inconsistent consensus outcomes whenever a voter sends a different value.
