## Substrate-honest gap note

The availability-preserving install pattern (read-old-into-memory → write-tmp → atomic-rename → side-car-backup) has its SAFE branches covered via dry-run + plutilLint tests. The UNSAFE branches (atomic-rename failure, backup-write failure) require either mocking `destPath` to a tmpdir or running the actual install and cleaning up — neither is straightforward without refactoring `main()` to accept a destPath override.
