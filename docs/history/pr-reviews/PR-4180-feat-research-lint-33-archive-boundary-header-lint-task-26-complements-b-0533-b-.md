## Description

## What

Adds `tools/research/lint-section-33-headers.ts` — small TS lint that checks every `docs/research/` file containing the `Archive scope (per GOVERNANCE §33)` marker for the 4 required header labels in the first 30 lines:

1. `Scope:`
2. `Attribution:`
3. `Operational status:`
4. `Non-fusion disclaimer:`

## Why

Empirical anchor: Codex P2 finding on PR [#4175](https://github.com/Lucent-Financial-Group/Zeta/pull/4175) (discriminating-falsifier file shipped with only `Scope` + `Attribution`; missing `Operational status` + `Non-fusion disclaimer`). Fixed manually + resolved no-op. This lint mechanizes the catch so future §33 archive files don't ship with missing labels.

Each catches a different recurring reviewer-finding class at a different scope.

Ran against all 374 `docs/research/` files: found exactly **1 pre-existing missing-header file** (`2026-05-14-shadow-session-close-off-duty-signaled-aaron.md`, missing all 4 labels). Out of scope for this PR (lint-tooling only); small follow-up to add the labels can land separately.
