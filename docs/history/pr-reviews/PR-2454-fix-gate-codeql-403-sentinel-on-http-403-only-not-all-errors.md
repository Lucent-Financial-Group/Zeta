## Description

## Summary

Follow-up to #2450. Codex reviewer (P2) caught that `ghApiOptional` silences **any** non-zero exit, not just HTTP 403 token-scope errors. The `code-scanning/default-setup` sentinel path would silently hide transient API failures, jq query errors, or endpoint regressions вАФ they'd get swallowed by the sentinel-stripping loop in `check-github-settings-drift.ts`.

- Adds `ghApiSkip403`: identical to `ghApiOptional` except it only returns `null` when `stderr` contains `"HTTP 403"`; all other errors re-throw and remain fatal
- Swaps `code-scanning/default-setup` to use `ghApiSkip403` so only the intended insufficient-token-scope 403 gets the sentinel
