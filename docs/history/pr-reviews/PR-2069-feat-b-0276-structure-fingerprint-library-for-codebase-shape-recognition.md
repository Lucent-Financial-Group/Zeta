  <pre>
  P0: `hasCycle` can report cycles in a DAG due to pushing the same node multiple times before it’s marked as `visiting`. In a graph like 0→1, 0→2, 2→1 (acyclic), node 1 can be pushed twice; the second pop hits `visiting.Contains node` and sets `found &lt;- true`. Fix by tracking a per-node state (unseen/visiting/visited) and marking `visiting` when a node is first discovered (before pushing), or by avoiding duplicate pushes with an explicit `discovered` set.

  </pre>

**Replies:**

  - **`AceHack` (human)** at 2026-05-08T12:26:13Z on `src/Core/StructureFingerprint.fs`:83 (association: MEMBER)

    <pre>
    Investigated — the original algorithm is actually correct for this case. The post-order marking (mark `visiting` on pop, not push) prevents the false positive: in a diamond DAG 0→1, 0→2, 2→1, node 1 gets pushed twice but only enters `visiting` when first popped; the second pop finds it in `visited` and skips.
