- Tracks two contract violations identified by Codex review on PR #2388:
  - `IntervalRing.Negate` violates `Negate a + a = Zero` for non-point intervals
  - `TropicalSemiring.Negate` throws at runtime despite interface contract advertising additive inverse

## Background

`ISemiring<'W>` documents the ring axiom `Negate a + a = Zero` as required for all implementations. Neither `IntervalRing` (for non-point intervals) nor `TropicalSemiring` satisfies this. The proposed design: split into `ISemiring` (no Negate, for tropical/probabilistic semirings) and `IRing : ISemiring` (with Negate, for integer/interval rings). DBSP retraction paths constrain to `IRing`.
