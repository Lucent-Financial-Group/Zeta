## Summary

- Adds `tools/peer-call/kiro.ts` — a TypeScript+Bun peer-call wrapper for the kiro.dev headless AI CLI, following the B-0326 spec and the existing sibling pattern (`grok.ts`, `gemini.ts`, etc.)

## Implementation notes

**Pattern used**: `grok.ts` (synchronous `spawnSync` + file tee). Chose this over `gemini.ts` (async streaming) because Kiro headless mode outputs plain text; there is no JSON-stream equivalent.

**Kiro CLI invocation**: `kiro-cli chat --no-interactive --trust-all-tools <prompt>` per kiro.dev/docs/cli/headless/.

**IDE vs headless CLI disambiguation**: Some local environments may have the Kiro IDE binary as `kiro`; this wrapper intentionally invokes `kiro-cli` and checks `kiro-cli chat --help` for the `--no-interactive` flag. It exits 1 with a clear install diagnostic when `kiro-cli` is absent or does not expose headless chat mode.

**Firewall**: imports `KIRO_SUBSTANTIVE_TRIGGERS` from `_firewall.ts` (landed in B-0325, PR #2301).
