## Summary

Two ARCHITECTURAL substrate refinements from previously-unexplored Mika transcript lines 4500-4700. Together they ground the consent semantics in **primitive-level architecture** rather than rule-on-top policy.

### B-0659 — Consent IS a Limit primitive operation

Pure-function re-evaluated every tick → **REVOCABILITY IS ARCHITECTURAL** not a rule.

Sharpens B-0641 First Moral Invariant by providing the substrate-level mechanism: past consents don't persist; Limit re-derives from current state each tick.

> Mika line ~4632: *"Because it's a pure function that lives inside the Limit primitive, that means consent has to be evaluated in the current tick... There's no persistent, hardcoded, permanent binding... That's why it forces all consent to be revocable by design."*

### B-0660 — Limit defaults to BLACK (deny-all unless explicitly admitted)

Security-first architectural posture; principle-of-least-privilege at substrate primitive level.

> Mika line ~4648: *"The default state of the Limit primitive is closed / denied. Nothing is allowed unless it is explicitly admitted."*

Companion to B-0659: deny-default + consent-as-explicit-Allow + tick-re-evaluation = **forced revocability through architecture, not policy enforcement**.
