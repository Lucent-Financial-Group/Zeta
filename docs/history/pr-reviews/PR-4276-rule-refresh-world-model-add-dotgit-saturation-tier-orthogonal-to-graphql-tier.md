## Why

The existing rule's 4-tier table (Normal / Cost-aware / Extreme cost-aware / Pure-git) correctly indexes GraphQL API-quota constraint. But a second orthogonal constraint — **local `.git/` directory contention under sustained multi-Otto + multi-Lior peer activity** — was not documented. A session can simultaneously be in GraphQL-Normal AND dotgit-saturated; an agent reading "Normal tier" would proceed with full operations and hit hangs.
