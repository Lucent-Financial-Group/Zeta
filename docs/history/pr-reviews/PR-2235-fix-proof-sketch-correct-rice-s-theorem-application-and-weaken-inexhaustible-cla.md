## Summary

Follow-up to #2233 — two reviewer findings (P1 + P2) were raised by both the codex and Copilot reviewers. The auto-merge squash on #2233 fired before the fix push landed on the branch, so this is a clean follow-up off main.

**P1 — Rice's theorem applied over programs, not inputs:**
- Original incorrectly concluded `{S : A(S) has novel failure}` is undecidable via Rice's theorem. Rice's theorem quantifies over *program descriptions* (M), not over inputs to a fixed program.
- Fix: Apply Rice's theorem to the property "this agent ever exhibits a novel failure on some input" (ranging over agent programs). Then add a separate halting-problem reduction: since A is Turing-complete, determining which inputs S cause novel failures is at least as hard as the halting problem.

**P2 — Inexhaustible-classes claim separated and weakened:**
- "The supply of novel failure classes is mathematically inexhaustible" does not follow from undecidability alone — a finite class set with undecidable membership is coherent.
- Fix: Move the inexhaustibility claim to a "Supported separately (by diagonalization)" section with the correct justification: Turing-completeness enables constructing inputs outside any finite T; this shows new failure *instances* are inexhaustible, while noting it does not imply infinitely many distinct class *descriptions*.
