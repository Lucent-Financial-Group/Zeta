## Architecture

| Layer | Path | Authority |
|---|---|---|
| Canonical | `tools/routines/<id>/` | git-tracked, PR-reviewed, diffable, shareable |
| Runtime | `~/.claude/scheduled-tasks/<id>/` | what Desktop reads at fire time |
| Installer | `tools/routines/install.ts` | idempotent repo → runtime sync |

Each routine: `SKILL.md` (prompt body + YAML frontmatter) + `schedule.json` (cron + metadata).

## Three-layer catch-43 defence

| Layer | Surface | Cadence | Failure mode covered |
|---|---|---|---|
| Session cron | CLI in-session | `* * * * *` | None — primary tick |
| Desktop routine | Persistent on disk | `0 */2 * * *` | CLI session death (12hr loss precedent) |
| `tools/routines/` repo | Git canonical | N/A | Maintainer-machine drift, runtime corruption |

Each layer is a different distance from the operative tick — closer = faster recovery, farther = more durable across substrate failure modes.
