## Operational learning — describe lint patterns WITHOUT triggering them

When authoring documentation about lint rules, use **descriptive prose** instead of **example code spans**. Example: instead of writing `` `PR ` `` in backticks to show the pattern, write "backtick-PR-space" or "PR followed by trailing space inside backticks" in plain prose. The lint engine treats backticks as code spans regardless of context.

This generalizes: any time documentation describes a pattern that triggers lint, the documentation itself can trigger the same lint. Use safe approximations.
