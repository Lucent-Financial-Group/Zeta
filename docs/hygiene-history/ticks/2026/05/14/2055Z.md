## Notes for future-Otto

Row close-out as a separate small PR is a clean pattern when the
substantive PR shipped without ticking DoD checkboxes. Reasons it's
worth keeping separate:

1. Substantive PR review focuses on the deliverable (not row metadata)
2. Row close-out fails-safe if substantive PR gets reverted (the
   close-out PR's diff is purely metadata; trivial to undo)
3. Backlog-index drift checker picks up the `status: closed` change
   on its next regen; cleaner audit trail

The pattern: substantive PR sets `open` → `in-progress`; close-out PR
sets `in-progress` → `closed` after the substantive PR merges.
