## Final coverage matrix (durable on main)

The helper now matches markdownlint MD032 across:

- **List markers**: `-` `+` `*` `1.` `1)` `12.` `12)` (capped 9 digits),
  bare markers (`-` alone)
- **List indent**: 0-3 spaces top-level; inside blockquote also 0-3
  spaces after the blockquote chain
- **Blockquote-context lists**: `> -`, `> 1.`, `> > -` nested,
  permissive `>[ \t]?` spacing
- **ATX headings**: `# Foo` and empty `##` form, NOT list-friendly
  (blank required after heading)
- **Fenced code blocks**: backtick + tilde; matching delim char +
  length; opener allows info string / closer must be whitespace-
  only; backtick info string forbids backticks (tilde allows tilde);
  inside-blockquote fences with content-indent accepted; fence
  transitions preserve list context for indented fences, terminate
  for flush-left / blockquoted top-level fences
- **YAML front matter**: skipped per markdownlint behavior
- **HTML comment regions** (`<!-- ... -->`): skipped including
  `markdownlint-disable` blocks
- **Markdownlint-cli2 ignore globs**: applied via JSONC parse in
  `--staged` mode
- **Lazy unindented list-item paragraph continuations**: don't end
  the list
- **Thematic break (`---`, `***`, `___`)**: terminates list (both
  top-level and blockquoted variants)
- **Blockquote-line after top-level list**: terminates the list
- **MD032 after-list side**: heading / thematic break / fence /
  blockquote directly after list (no blank) fires MD032
- **Staged-blob reads**: `--staged` reads index via `git show :path`,
  not working tree; throws loud on git failure
