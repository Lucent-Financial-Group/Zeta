## Round-13 pattern

Round 13 was an operational-completeness finding at the very
foundation of `--staged` mode: was the helper actually checking
what CI checks? The answer was "no, it was checking what's on disk
right now," which can diverge from the staged version after an
edit-then-no-restage cycle. The fix routes through `git show :path`
to read the index directly.

This is the kind of finding that's only catchable by a careful
reviewer thinking about the operational lifecycle (stage → edit →
push), not just the algorithm. The coverage matrix in the PR
description now includes:
