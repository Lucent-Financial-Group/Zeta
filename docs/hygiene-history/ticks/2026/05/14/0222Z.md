## Round-14 pattern

Round 14 was breadth-of-coverage refinements:

- Blockquote spacing: the regex was too strict (`>\s?` = 0-1 space).
  CommonMark allows any positive number of spaces. Three call sites
  (`isListItemStart`, `isBlank`, `isHeading`) fixed in lockstep.
- Fence-after-blockquote: round 12 over-corrected by treating
  `[ \t>]` symmetrically. The CommonMark grammar is asymmetric ‚Äî
  whitespace-indent is list-item-body, `>` is a new block context.
- Staged-blob I/O: round 13 introduced `checkStagedFiles` with a
  silent-skip-on-failure inherited from round 2's filesystem
  discipline. That was wrong ‚Äî staged blob reads have different
  semantics (the path is staged for commit; can't-read is anomaly).
- Lint discipline: each `spawnSync("git", ...)` in the test file
  needed suppression; consolidated into a single helper.
