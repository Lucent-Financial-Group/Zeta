## Round-15 pattern

Round 15 caught the long tail of CommonMark list-termination rules:

- Blank line (round 2)
- ATX heading (round 12)
- Top-level fence (round 12)
- Top-level blockquoted fence (round 14)
- **Thematic break (round 15)**
- **Top-level blockquote line (round 15)**

The `inList` state had to be lifted from boolean to a 3-state enum
(`false | "top" | "block"`) to handle the asymmetry: blockquote
terminates top-level lists but continues blockquoted ones. Same
classification will be useful for future rounds if other
block-types-inside-blockquote cases surface.
