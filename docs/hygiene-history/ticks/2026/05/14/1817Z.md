## Notes for future-Otto

**Stash-pop hazard pattern**: when `git stash push <file>` fails (e.g., untracked file without `--include-untracked`), it does NOT touch the stash stack — but if a subsequent command implicitly triggers a stash pop (e.g., `git checkout -b` from a state that conflicts with the top stash, OR a misread of a partial command), the top stash gets applied with conflicts. Diagnose by checking for unexpected modifications + `<<<<<<<` markers immediately after branch operations.

**Worktree-prune friction-reducer**: `git worktree prune --expire=now -v` is the single most useful command for cleaning multi-Otto session-crash leftovers. The default prune (no `--expire`) skips entries below a minimum age. The `--expire=now` flag bypasses the floor.
