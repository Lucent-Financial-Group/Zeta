## Notes for future-Otto

**Reference classification taxonomy** developing across batches:

1. **Concrete file ref**: backtick'd path; should resolve via `test -e`
2. **Glob ref**: pattern like `feedback_rule_number_*`; resolves via `ls $pat | wc -l >= 1`
3. **Template path**: `<placeholder>/<name>.md`; should resolve to ≥1 instance in the named surface
4. **Backlog ID**: `B-NNNN`; map to file via `ls docs/backlog/P*/B-NNNN-*.md`
5. **Legacy-noted ref**: rule explicitly says "NOT to use" — file may or may not exist; both states are healthy depending on rule semantics
6. **Transient/ephemeral ref**: rule says "DON'T rely on this" — MISS is expected
7. **Anti-pattern ref**: rule mentions for contrast (e.g., "the old way was X, now use Y")

The audit logic must distinguish stale-pointer (real bug) from these healthy patterns. Doing this by-hand per batch is working but mechanization becomes more valuable as the rule corpus grows.
