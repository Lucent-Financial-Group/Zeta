## Notes for future-Otto

The `EXCLUDE_RE` regex in `audit-machine-specific-content.ts` is the
single line that gates exclusion. The docstring comment above it now
explains WHY each path-class is excluded — so future contributors
considering "should this also be excluded?" have a precedent + a
discriminator (the maintainer-note + maintainer-only-artifact framing).

Pattern for future audit-exclusion decisions:

1. Is the file's content **canonically machine-specific by purpose**? (Plists are; bash scripts and SKILL.md are NOT.)
2. Does the file carry a maintainer-note comment explaining why? (Discriminator.)
3. Is the file a maintainer-only artifact (not consumed by portable substrate)?

All three must be YES to justify exclusion. Otherwise the flagging IS
the signal — a real gap that needs portability work.
