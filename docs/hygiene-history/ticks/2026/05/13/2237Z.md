## Reviewer-thread metabolism observation

Codex round-2 caught a subtle bug in my round-1 fix: rejecting
`flags.worktree === "true"` unconditionally also rejects the legitimate
`--worktree true` invocation. The root cause was that parseArgs used
the string `"true"` as both the bare-flag sentinel AND a possible
explicit user value — ambiguous. The right fix was to make the
sentinel a different TYPE (boolean `true`) so the two cases are
unambiguous at compile time, not at runtime by string-compare. This
is the kind of design-improvement reviewer-thread work that
substrate-honest review produces — round-2 was a better fix than
round-1, not just a patch on top.
