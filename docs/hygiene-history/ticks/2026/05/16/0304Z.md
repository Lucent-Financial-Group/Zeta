## The 2 new audit findings — both real, both cross-platform

### P1 — directory inputs bypass `--files` validation

Tick 9's fixup added `existsSync` validation to `--files`, but a directory or unreadable file passes `existsSync` and then `readFileSync` throws `EISDIR`/`EACCES` inside `extractLinks` — bypassing the documented exit-64 contract.

Fix: also require `statSync(abs).isFile()`, with stat wrapped in try/catch for permission failures.

Verify:

```text
--files docs/hygiene-history/   → "input not a regular file" + exit 64
--files /tmp/does-not-exist     → "input not found" + exit 64
```

### P2 — `ROOT + "/"` prefix check breaks on Windows

`abs.startsWith(ROOT + "/")` hardcoded the POSIX separator. On Windows, `resolve()` returns paths with backslash, so the prefix `C:\repo/` never matches valid in-repo targets like `C:\repo\docs\...`. Under `--enforce` mode in Windows CI, every valid link would be flagged as `escapes-repo` — total false-positive sweep.

Fix: `import { sep as PATH_SEP } from "node:path"` + `ROOT + PATH_SEP`. Platform-correct.
