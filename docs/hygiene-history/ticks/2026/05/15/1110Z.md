### Meta-meta finding: illustration-of-bug = instance-of-bug

The self-inconsistency error has now recurred THREE consecutive ticks (1050Z, 1056Z, 1103Z). Pattern: I document the discipline + illustrate the bug using the concrete bare-number form + Copilot flags the illustration AS a violation.

The discipline says "do not use bare hash-NNNN in body text". When illustrating the violation, the natural choice is to QUOTE the bug as concrete bare-NNNN (e.g., backtick-hash-3420-backtick). But that concrete form IS the bug.

**Real mitigation**: use placeholders for illustrations of the bare-NNNN form. Acceptable substitutes:

- Word-form: `the bare hash-NNNN form`
- Asterisk-form: backtick-hash-3-asterisk-asterisk-asterisk-backtick
- Generic-letter-form: backtick-hash-XXXX-backtick
- Meta-description: `prefix-less reference to a PR by number alone`

The corrected grep pattern (per Codex + Copilot on PR #3425): `grep -nE "(^|[^P]) #[0-9]{4}|^#[0-9]{4}"` requires non-P character before space-#.
