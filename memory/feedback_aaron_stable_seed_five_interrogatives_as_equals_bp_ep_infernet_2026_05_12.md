### What BP/EP message-passing is

- **BP (Belief Propagation)**: algorithm for inference
  on probabilistic graphical models; messages pass along
  edges of a factor graph; each node updates its belief
  based on incoming messages
- **EP (Expectation Propagation)**: generalization of BP
  to non-conjugate distributions; works with moment-
  matching rather than exact updates
- **Infer.NET**: Microsoft Research's probabilistic
  programming framework supporting both BP and EP

### Why the seed matters for BP/EP

BP/EP message-passing requires an initial assignment of
beliefs (the **seed**). The algorithm then iteratively
updates messages until convergence. Convergence
properties depend on seed quality:

- **Stable seed** → algorithm converges to a useful
  equilibrium
- **Unstable seed** (some dimensions suppressed,
  asymmetric) → algorithm may oscillate, converge
  to a degenerate equilibrium, or fail to converge
- **Symmetric seed across all relevant interrogative
  dimensions** → maximal convergence stability

If the interrogative seed only contains 4 of 5
interrogatives (WHY suppressed), the BP/EP graph misses
the WHY-edges. Message-passing on a graph missing edges
can't propagate beliefs through the missing region —
those beliefs become orphaned or incoherent.

**The stable seed = all interrogatives represented =
BP/EP graph complete = convergent inference.**
