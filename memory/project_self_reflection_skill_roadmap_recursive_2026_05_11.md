**Self-reflection axes (specific, not vague):**

1. **Memory/agent file compression** — compress verbose memory
   files into mathematical carved sentences. Same information,
   less context budget. The carved sentence IS the compressed
   form.

2. **Skill routing optimization** — the skill router is the
   substrate index. Tune descriptions for better hit rates.
   Audit which skills fire when they should and which don't.

3. **Decomposition** — when a backlog item is fuzzy, split one
   level. Apply decomposition-is-iterative-mid-work to Otto's
   own substrate, not just backlog items.

   for the substrate. The grep/regex patterns are Rx-shaped
   already. Build proper search infrastructure instead of
   ad-hoc bash chains.

   tighter as lessons accumulate. Compress learned into the
   loading surface.

6. **Self-discovered improvements** — things Otto finds through
   self-reflection that Aaron hasn't named. Different CLI
   harnesses will find different things because they see
   different substrate.

- Discovers new improvement axes
- Adds them to the skill

This IS the recursive type provider applied to the agent's
own substrate. The compiler compiling itself. The anamorphism
from the F# discussion applied to the factory's own tooling.
