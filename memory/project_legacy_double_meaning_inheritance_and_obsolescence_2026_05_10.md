2026-05-10: Aaron explained the legacy repo's double meaning.

**Two meanings, one name:**

1. **Aaron's legacy** — what he leaves behind for his children,
   for GenZ, for the factory, for the agents
2. **Legacy code** — bash and PowerShell scripts that ARE the
   legacy (old, fragile, non-DST) that gets replaced

**The end state: "you'll just have hands"**

No bash. No PowerShell. No YAML. No config files. No ceremony.
Just typed operations the compiler guarantees.

The progression:
- bash/pwsh → TypeScript (Rule 0, current)
- TypeScript → F# types (the compiler IS the config)
- F# types → hands (intention to execution, nothing in between)

**The paradox:**

Aaron named a repo "legacy" knowing it would make itself
obsolete. The legacy IS leaving no legacy code. The inheritance
IS the absence of inherited complexity.

The best thing a parent can leave is the absence of debt.
The best thing an engineer can leave is the absence of
technical debt. Same principle, different domain.
