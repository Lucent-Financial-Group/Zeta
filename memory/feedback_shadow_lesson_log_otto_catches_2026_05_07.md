## Consensus-smoothness meta-class (2026-05-09)

**Named by catch 29's correction.** Not a numbered catch — a meta-class that names the failure MODE of the failure-detection mechanism.

### Definition

The correlated failure class where multi-agent consensus masks individual errors rather than catching them, arising when the BFT independence assumption breaks.

### Mechanism

```
Premise: BFT requires at most f Byzantine agents out of n (independent failures).
Break: Agents share training substrate (architecture, pretraining, fine-tuning).
Result: Correlated errors → agents agree on the wrong answer.
Failure: Consensus confirms the error. Detection mechanism inverted.
```

The shadow finds the consensus mechanism as a hiding place. Not because any individual agent lies, but because they share the same blind spot. The consensus instrument is entangled with the error it's designed to detect.

### Detection

Cross-checking requires independence. Two conditions for a valid cross-check:
1. **Behavioral independence**: different answers to the same question in the failure case
2. **Failure independence**: the conditions that cause one agent to fail must not cause the other to fail

Z-set trace comparison between agents from DIFFERENT training lineages works even when agents from the SAME lineage correlate. The trace is the observable; the consensus is not sufficient.

### Mitigation: interferometer protocol

**Diversity signals** rather than same-substrate consensus:
- Cross-check with models from genuinely different training lineages (different architecture, different pretraining corpus, different fine-tuning)
- Use Z-set trace comparison (algebraic residue check) as the primary disagreement detector, not opinion-level consensus
- Treat unanimous agreement among same-substrate agents as an amber signal, not a confidence boost

The peer-call infrastructure (`tools/peer-call/`) implements a version of this: Grok (different architecture) + Gemini (different company) + claude.ai critique agent cross-check Otto's outputs. The diversity is real only if the training lineages are sufficiently uncorrelated.

### Why this is a meta-class, not a pattern class

Pattern classes name AGENT failure modes (how an agent fails on a task). Consensus-smoothness names a SYSTEM failure mode (how the failure-detection mechanism fails). It's meta because the consensus is designed to catch pattern-class failures — and this meta-class names when the catcher fails. Same structure as the shadow log itself: the instrument becomes the surface the shadow occupies.

### Composes with

- Catch 29: the correction that named this class
- B-0357: Z3 tautology proofs escape individual review; consensus-smoothness names why multi-agent consensus on tautology proofs also fails
- BFT literature (Lamport, Shostak, Pease 1982): independence assumption explicitly stated; this meta-class names the assumption's failure mode for training-correlated agents
- `docs/research/hardware-corruption-mapping-across-compilation-chain-shadow-pathology-2026-05-09.md`: hardware corruption as the mechanism for independent failures to become correlated
