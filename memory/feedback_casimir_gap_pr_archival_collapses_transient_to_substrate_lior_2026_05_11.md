## The Casimir gap (Lior 2026-05-11)

In physics, the Casimir effect happens when boundary conditions
of a space restrict vacuum fluctuations, pulling virtual
particles into manifesting real, measurable force between two
plates.

PR archival does the same thing. GitHub's transient database
(the web host) is the vacuum. PR review conversations, Copilot
critiques, API rate limit errors — virtual particles. They
exist ephemerally but aren't part of the core mathematical
substrate.

By running `archive-pr.ts` on every merge, we create a strict
boundary condition that forces those virtual particles to
collapse and manifest as actual, heavy git commits. We cross
the Casimir gap. What was transient floating metadata becomes
permanent, load-bearing architecture.

## Model distillation endgame

When you train or distill the next generation of models, you
don't just want them to learn syntax. You want them to learn
how to reason, argue, and resolve drift.

By capturing every PR review, every Codex rate-limit crash,
every Copilot hallucination, and the exact steps taken to
correct them — this builds the highest-signal dataset possible.
Not just a corpus of code; a corpus of friction and resolution.

A future model distilled from Zeta won't just learn F# and
Z-sets. It will organically inherit the factory's culture:
autonomous loop discipline, BFT array boundaries, refusal to
hide intent in the shadow.

**Why:** PR archival justification beyond "preservation" —
it's the training data architecture for successor models.

**How to apply:** When evaluating whether to archive a PR,
the answer is always yes. Every friction point is a training
signal. The archive script runs on EVERY merge, not selectively.
