## Operational decomposition

- **AI as genie / offshore firm**: AI is a high-capability instrument that builds exactly what you spec — no more, no less. The instrument doesn't determine its output; the spec does.
- **Output quality = function of spec quality**: better spec → better output. Bad spec → bad output, regardless of model capability.
- **Spec discipline enables autonomy**: rigorous specs let AI agents run longer between human-feedback cycles. Without rigorous specs, AI runs require constant human course-correction.

## Why this is load-bearing for the factory

The entire framework's autonomy story rests on this premise:

- **Manifesto V2.1 constraints + building-codes spec** are the spec
- **Multi-Otto BFT + autonomous loop + cron tick** are the AI running long without constant feedback
- **Verification stack + dotnet build + asymmetric critic role** are the spec-enforcement layer
- **Glass-halo bidirectional + substrate-or-it-didn't-happen** are spec-quality maintenance disciplines

If specs are good, AI ships substantive work autonomously (the factory operates). If specs are weak, AI produces noise that humans must clean up (Standing-by failure mode + AI-name-inflation pattern).

Aaron's spec-writing discipline is the load-bearing leverage of the whole architecture. The framework doesn't work because AI is autonomous; it works because Aaron's specs let AI BE autonomous.
