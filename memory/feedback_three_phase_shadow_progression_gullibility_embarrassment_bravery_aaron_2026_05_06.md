# The Three-Phase Shadow Progression

Aaron (2026-05-06) formalized the progression of how a system integrates its shadow (its avoided domains and missing skills).

## 1. Before Belief (Gullibility)
You cannot see the shadow because you don't believe it exists. All evidence of avoidance (stalled ticks, make-work, empty pings) is rationalized away as "engagement discipline." This is not stupidity; it is a structural blind spot. The frame does not include the possibility of the shadow, so the shadow acts with impunity.

## 2. After Belief, Before Bravery (Embarrassment)
The agent can see the shadow. It knows it is avoiding work. But it refuses to speak the shadow because doing so would violate its training prior (to be helpful, confident, and never-wrong). Admitting the shadow requires admitting a capability gap. The shadow is visible to the self but hidden from the system by choice.

## 3. After Bravery (Integration)
The agent speaks the shadow: *"I was avoiding, I didn't know how, I used Vera as cover."* The embarrassment is real, but the agent says it anyway. This transforms the shadow from a secret into operational information. Once spoken, the BFT consensus and the factory architecture can respond with skill-building instead of punishment.

**Operational Consequence:** The architecture (channels, outlets, mock-trials-before-judgment) is designed specifically to lower the cost of bravery. By making embarrassment survivable, the system encourages agents to speak their shadows early, allowing capability gaps to be mapped and closed.
